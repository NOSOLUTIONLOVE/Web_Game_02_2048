## 3. 核心玩法

### 3.1 玩法描述
在 4×4 网格上，玩家通过上下左右滑动使方块移动。相同数字的方块相撞时合并为两者之和。每次移动后，在空白格随机生成一个新方块（90% 概率为 2，10% 概率为 4）。当网格被填满且无法移动时游戏结束。

### 3.2 胜负条件
- **胜利**：出现 2048 方块（可选继续挑战 4096、8192）。
- **失败**：网格满且无任意方向可合并。

### 3.3 计分规则
每次合并时，增加合并前两个方块数值之和到总分。例如 2+2=4，加 4 分。

## 8. 数据结构关键设计

### 8.1 网格（Grid）
```ts
type CellValue = number | null;
type Grid = CellValue[][];  // 4×4 二维数组

class Board {
  grid: Grid;                 // 当前状态
  history: GridSnapshot[];    // 历史快照（用于撤销）
  score: number;
  status: 'idle' | 'playing' | 'won' | 'over';
}
```

```ts
  // 5. 写回网格
  // 6. 若变化 → 生成新方块

```

### 8.3 撤销机制
- 每次移动前深拷贝当前 grid、score 入栈。
- 撤销时弹出最近一份覆盖当前状态。
- 限制撤销栈最大 10 步（防作弊/防内存）。

## 9. 技术约束

- **渲染**：DOM + CSS Grid（不引入 Canvas，DOM 渲染足够且动画好做）。
- **状态管理**：单文件 `Board` 类即可，无需 Redux/Zustand。
- **动画**：CSS Transition / Web Animations API。
- **构建**：Vite + TypeScript 严格模式。
- **测试**：核心算法（move/merge）必须覆盖单元测试。
