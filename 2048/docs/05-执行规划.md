## 二、总体策略

### 2.1 核心策略：分阶段交付

继承 snake 的 P0-P5 阶段模式，本项目实际执行合并为 A-F 六个阶段：

```
A 脚手架 ──► B 引擎 ──► C 状态 ──► D UI ──► E 动效 ──► F 验收
  1h           2h         0.5h        3h        1.5h        1h
```

### 2.2 阶段依赖关系

```
A（必须最先）
  │
  ├──► B ──► C ═══► D ═══► E ═══► F
       │     │       │       │       │
       │     │       │       │       └──► 部署上线
       │     │       │       └──► 体验打磨
       │     │       └──► 完整 UI
       │     └──► store 桥接
       └──► 核心算法 + 单测
```

## 四、关键风险与应对

### 风险 1：触屏滑动与点击冲突

| 概率 | 影响 | 应对 |
| --- | --- | --- |
| 中 | 中 | `touchmove` 阻止默认滚动，`touchend` 计算方向 |

### 风险 2：Framer Motion layout 性能

| 概率 | 影响 | 应对 |
| --- | --- | --- |
| 低 | 中 | 4×4 最多 16 个 tile，性能完全够 |

### 风险 3：pnpm 11 build script 策略

| 概率 | 影响 | 应对 |
| --- | --- | --- |
| 高 | 中 | 创建 `pnpm-workspace.yaml` + `onlyBuiltDependencies: [esbuild]` |

### 风险 4：iOS Safari AudioContext 限制

| 概率 | 影响 | 应对 |
| --- | --- | --- |
| 高 | 中 | `engine.start()` 中调 `audio.resume()` 必须在 user-gesture 后 |

### 风险 5：WinModal 阻断用户

| 概率 | 影响 | 应对 |
| --- | --- | --- |
| 中 | 中 | 用普通 div 不用 Dialog，保留键盘方向键 |

## 十一、v3.0 全面深度完善与优化

> 2048 从 v0.1.0 到 v3.0 的全面升级

| 字段 | 内容 |
| --- | --- |
| 文档版本 | v3.0 |
| 创建日期 | 2026-06-24 |
| 当前状态 | ✅ 已完成 |

### 11.1 v3.0 升级目标

将 2048 从"可玩的原型"提升为"打磨完善的精品小游戏"，在性能、玩法深度、视觉表现、个性化、数据统计与可访问性六个维度全面进阶。

### 11.2 v3.0 新增功能

| 维度 | 功能 |
| --- | --- |
| 性能修复 | 稳定 Tile key 连续动画、lastMove 响应式订阅、响应式棋盘、后台标签页输入锁定 |
| 玩法深度 | 4×4/5×5/6×6 网格尺寸、开局 3-2-1-GO 倒计时、合并连击 Combo、撤销按钮可视化 |
| 视觉反馈 | 合并粒子爆裂、分数飘字、屏幕震动、超级方块配色（4096-65536）、合并发光 |
| 个性化 | 三套主题（经典暖色/暗夜霓虹/纯净极简）、网格尺寸选择、音量滑块、撤销深度可配 |
| 数据统计 | 累计游戏次数/步数/最高方块/各模式最高分、主菜单统计卡片 |
| 代码质量 | React ErrorBoundary、ARIA live region、键盘导航、代码瑕疵修复 |

### 11.3 v3.0 技术要点

- **Tile id 追踪**：Board 引擎为每个方块分配稳定唯一 id，Tile 组件基于 id 作为 React key，Framer Motion layout 动画连续不中断
- **可变网格**：Board 构造函数接收 size + winValue，支持 4/5/6 三种尺寸，WIN_VALUE 动态计算
- **连击系统**：MoveResult.combo 记录合并组数，GameEngine 按 1.5×/2×/3× 倍率追加 bonus 分
- **主题系统**：CSS 变量 + CONFIG.THEMES 配色表，根元素 data-theme 属性切换
- **localStorage v2 迁移**：persist version 2 + migrate 函数，旧数据自动补充新字段
