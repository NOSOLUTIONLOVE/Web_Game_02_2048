## 一、架构总览

### 1.1 架构原则

| 原则 | 说明 |
| --- | --- |
| **三层严格分离** | engine 零 React 依赖，state 用 Zustand，UI 只读 store + 调 engine |
| **单向数据流** | 用户输入 → engine → store → UI |
| **状态机** | 5 个游戏阶段显式枚举，避免散落的布尔标志 |
| **公开回调** | engine 通过 callbacks 通知 store，UI 通过 engine 公开方法触发操作 |
| **零全局变量** | 所有状态封装在类实例或 store 中 |

### 1.2 三层架构图

```
┌─────────────────────────────────────────────────┐
│ UI 层（React 组件）                              │
│   ├─ 2048Game（挂载点 + Context）               │
│   ├─ Board / Tile / HUD / Footer                │
│   ├─ Overlays（MainMenu / Pause / GameOver / Win）│
│   └─ SettingsPanel                              │
│        │   ▲                                    │
│        │   │ useGameStore（订阅）                │
│        ▼   │                                    │
├─────────────────────────────────────────────────┤
│ State 层（Zustand store）                       │
│   ├─ phase / grid / score / moves               │
│   ├─ highScore（持久化）                        │
│   ├─ audioEnabled（持久化）                     │
│   └─ applyBoard / setPhase / setNewRecord       │
│        │   ▲                                    │
│        │   │ 回调                               │
│        ▼   │                                    │
├─────────────────────────────────────────────────┤
│ Engine 层（纯 TS 类）                            │
│   ├─ GameEngine（编排 + 状态机）                │
│   ├─ Board（网格 / 移动 / 合并 / 撤销）         │
│   ├─ Input（键盘 + 触屏）                       │
│   └─ AudioSystem（Web Audio 合成）              │
└─────────────────────────────────────────────────┘
        │   ▲
        │   │ DOM 事件 / 玩家操作
        ▼   │
       浏览器
```

## 三、模块详解

### 3.1 Engine 层

#### `engine/Board.ts` —— 网格核心 ⭐⭐⭐

**职责**：
- 网格状态管理（4×4 `CellValue[][]`）
- 移动 + 合并算法
- 新方块生成（90% 概率 2，10% 概率 4）
- 撤销栈（深度 ≤ 10）
- 胜负判定

**关键 API**：
```ts
export class Board {
  public getGrid(): Grid;
  public getScore(): number;
  public getMoves(): number;
  public getStatus(): BoardStatus;
  public canUndo(): boolean;
  public reset(): void;
  public move(direction: Direction): MoveResult;
  public undo(): boolean;
  public keepPlayingAfterWin(): void;
}
```

**移动算法（旋转到 left 统一处理）**：
1. 根据方向旋转网格
   - left: 保持
   - right: 水平翻转
   - up: 逆时针旋转 90°
   - down: 顺时针旋转 90°
2. 对每行执行"向左合并"（压缩 → 合并相邻 → 填 null）
3. 反向旋转回原坐标系
4. 用 `toOriginalCoords()` 把合并位置转回原坐标

---

#### `engine/GameEngine.ts` —— 编排层

**职责**：
- 桥接 Board + Input + Audio
- 5 阶段状态机
- 通过 callbacks 通知 store

**状态机**：
```
menu ──confirm──▶ playing ──pause──▶ paused ──pause──▶ playing
                       │                                  │
                       ├─won──▶ won ──keepPlaying──▶ playing
                       │              ──confirm──▶ playing
                       └─over──▶ over ──confirm──▶ playing
```

**公开方法**：
```ts
export class GameEngine {
  constructor(callbacks: GameEngineCallbacks);
  public start(): void;
  public stop(): void;
  public startGame(): void;
  public togglePause(): void;
  public undo(): void;
  public keepPlaying(): void;
  public backToMenu(): void;
  public handleMove(dir: Direction): void;  // 公开，触屏绑定调用
  public getAudio(): AudioSystem;
  public getInput(): Input;                  // 公开，Board 触屏绑定调用
}
```

---

#### `engine/Input.ts` —— 输入系统

**职责**：
- 键盘 + 触屏
- 阈值过滤
- 通过 callbacks 通知 engine

**键盘映射**：

| 按键 | 行为 |
| --- | --- |
| ↑ ↓ ← → / W A S D | 移动 |
| Z | 撤销 |
| R | 重置 |
| P / Esc | 暂停 |
| Space / Enter | 确认（菜单 / 结束） |

**触屏逻辑**：
- 记录 touchstart 坐标
- touchend 时计算 dx/dy，取绝对值较大者方向
- 距离 ≥ 30px 才触发（阈值防误触）

---

#### `lib/audio.ts` —— 音效系统

**职责**：
- Web Audio API 合成 6 种音效
- 懒加载 AudioContext（iOS Safari 兼容）
- 启用 / 禁用开关

**6 种音效**：

| 事件 | 合成方式 | 持续 |
| --- | --- | --- |
| move | sine 800→400Hz | 0.05s |
| merge | triangle 200+value×8 → 0.5× | 0.12s |
| spawn | sine 1200→600Hz | 0.03s |
| win | sine C5-E5-G5 三音 | 0.3s×3 |
| lose | sawtooth 440→110Hz | 0.4s |
| click | sine 600Hz | 0.04s |

---

### 3.2 State 层

#### `store/useGameStore.ts` —— Zustand store

**职责**：
- 桥接 UI 与 GameEngine
- 持久化 highScore + audioEnabled
- 提供订阅接口给 UI

**状态字段**：
```ts
interface GameStore {
  phase: GamePhase;       // 'menu' | 'playing' | 'paused' | 'won' | 'over'
  grid: Grid;             // 4×4 CellValue[][]
  score: number;          // 当前局分数
  highScore: number;      // 历史最高（持久化）
  moves: number;          // 步数
  isNewRecord: boolean;   // 破纪录标识
  audioEnabled: boolean;  // 音效开关（持久化）
  canUndo: boolean;       // 是否可撤销
  lastMove: MoveResult | null; // 最近一次移动（驱动动画）
}
```

**持久化**（`zustand/middleware/persist`）：
```ts
partialize: (s) => ({ highScore: s.highScore, audioEnabled: s.audioEnabled })
```

### 3.3 UI 层

#### `components/2048Game.tsx` —— 挂载点 ⭐⭐⭐

**职责**：
- 实例化 GameEngine（useEffect）
- 同步 audioEnabled 到 engine
- 用 Context 暴露 engine 给子组件
- 渲染 HUD + Board + Overlays

**Context API**：
```ts
export const EngineContext = createContext<GameEngine | null>(null);
export function useEngine(): GameEngine;
```

**布局**：
```tsx
<EngineContext.Provider value={engine}>
  <HUD />
  <div className="relative">
    <Board />
    {phase !== 'playing' && <Overlays />}
  </div>
</EngineContext.Provider>
```

---

#### `components/Board.tsx` —— 网格容器

**职责**：
- 4×4 CSS Grid 16 个空格槽
- 浮层渲染所有 Tile
- 触屏手势绑定到本 DOM 节点

**关键代码**：
```tsx
useEffect(() => {
  const el = boardRef.current;
  if (!el || !engine) return;
  const input = engine.getInput();
  input.bind({ onMove, onUndo, onReset, onPause, onConfirm }, el);
  return () => input.unbind();
}, [engine]);
```

---

#### `components/Tile.tsx` —— 单方块

**职责**：
- 渲染一个非空方块
- 配色（CONFIG.COLORS.TILES）
- Framer Motion 动画

**核心动画**：
```tsx
<motion.div
  layout
  initial={isNew ? { scale: 0 } : false}
  animate={{ scale: isMerged ? [1, 1.2, 1] : 1, top, left }}
  transition={{ type: 'spring', stiffness: 500, damping: 35 }}
/>
```

**动画触发条件**：
- `isNew`: scale 0 → 1 出现
- `isMerged`: scale 1 → 1.2 → 1 弹动
- 移动：Framer Motion `layout` 自动从旧 top/left 过渡到新 top/left

---

#### `components/HUD.tsx` —— 顶栏

**结构**：
```
┌─────────────────────────────────────────────────┐
│ 2048  [Score] [Best] [Steps] [NEW!] [🔊] [⚙]   │
└─────────────────────────────────────────────────┘
```

**数字动效**：
```tsx
<motion.div
  key={value}
  animate={{ scale: [1, 1.25, 1] }}
  transition={{ duration: 0.3, ease: 'easeOut' }}
>
  {value}
</motion.div>
```

---

#### `components/Overlays.tsx` —— 状态分发

```tsx
{phase === 'menu' && <MainMenu />}
{phase === 'paused' && <PauseOverlay />}
{phase === 'over' && <GameOverModal />}
{phase === 'won' && <WinModal />}
```

**Modal 类型选择**：
| Phase | 组件 | 阻塞输入？ | 原因 |
| --- | --- | :---: | --- |
| menu | MainMenu（div） | ✅ | 等待用户点开始 |
| paused | PauseOverlay（div） | ✅ | 等待用户点继续 |
| over | GameOverModal（Dialog） | ✅ | 等待用户点重开 |
| won | WinModal（div） | ❌ | 不阻断，玩家可继续玩 |

## 五、生命周期

```
main.tsx
  └─► ReactDOM.createRoot(...).render(<App />)

  └─► App
        └─► <Game2048 />
              └─► useEffect：new GameEngine(callbacks)
                    ├─ new Board()
                    ├─ new Input().bind(cbs, window)
                    └─ new AudioSystem()
              └─► e.start()
                    ├─ audio.resume()（iOS 兼容）
                    └─ notifyBoardChange()（触发首次渲染）

  玩家按方向键
    └─► Input.onMove(dir)
          └─► GameEngine.handleMoveInternal(dir)
                ├─ Board.move(dir) → MoveResult
                ├─ AudioSystem.playMove/playMerge/playSpawn
                ├─ storage.set(高分)
                ├─ callbacks.onNewRecord(高分)
                └─ callbacks.onBoardChange(snapshot)
                      └─► useGameStore.applyBoard(snapshot)

  玩家关闭页面
    └─► useEffect cleanup：e.stop()
          └─► Input.unbind()
```

## 六、设计模式应用

| 模式 | 应用位置 | 作用 |
| --- | --- | --- |
| **三层架构** | engine / state / UI | 关注点分离 |
| **状态机** | GameEngine.phase | 5 个游戏阶段显式管理 |
| **观察者** | engine callbacks | 解耦 engine 与 store |
| **单例** | `storage` | 无状态工具类 |
| **Context** | EngineContext | engine 实例跨组件共享 |
| **策略模式** | CONFIG | 运行时参数化 |
| **不可变风格** | Board.move 返回新 grid | 便于回退 + 测试 |

## 七、与 snake 项目架构对比

| 维度 | snake | 2048 |
| --- | --- | --- |
| 渲染 | Canvas 2D | DOM + CSS Grid |
| 输入绑定位置 | Game 类内自动 | Board.tsx 的 useEffect |
| 状态机 | 4 阶段（menu/playing/paused/over） | 5 阶段（多 won） |
| 状态层 | Zustand（同） | Zustand（同） |
| 动画 | Canvas 帧循环内绘制 | Framer Motion + layout |
| 主循环 | requestAnimationFrame | **无**（输入驱动） |
| 撤销 | 无 | 有（10 步） |
| 步数统计 | 无 | 有 |

**复用部分**：
- shadcn/ui 组件库
- Tailwind 配置（暗黑紫调色板）
- AudioSystem 模式
- storage 工具
- GameEngine 编排模式

## 八、测试策略

### 8.1 单元测试

**Board.test.ts**（16 个测试）：
- moveRowLeft 核心算法
  - 初始状态
  - reset 后 2 个方块
  - 向左合并（2+2=4, 4+4+4+4=8+8, 2+2+4+4=4+8）
