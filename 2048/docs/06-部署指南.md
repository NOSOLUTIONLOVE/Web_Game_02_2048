## 二、方案 A：Vercel 部署（推荐）

### A.1 通过 Vercel 网站（最简单）

**前置**：GitHub 账号 + Vercel 账号（用 GitHub 登录即可）

#### 步骤

1. **推送代码到 GitHub**

   ```bash
   cd /Users/carlshen/Documents/Project/Web_Game_01
   git add .
   git commit -m "feat: 2048 v0.1.0 - 完整可玩 + 动画 + 撤销 + 步数"
   git push origin main
   ```

2. **在 Vercel 导入项目**
   - 访问 https://vercel.com/new
   - 选择 **Import Git Repository**
   - 找到 `Web_Game_01` 仓库，点击 **Import**

3. **配置项目**
   - **Framework Preset**：自动识别为 `Vite`
   - **Root Directory**：`games/2048`（重要！不是仓库根目录）
   - **Build Command**：`pnpm build`（默认）
   - **Output Directory**：`dist`（默认）
   - **Install Command**：`pnpm install`（默认）

4. **部署**
   - 点击 **Deploy** 按钮
   - 等待 1-2 分钟
   - 部署成功后会显示 🎉 和一个 URL，形如 `https://web-game-01.vercel.app`

5. **（可选）绑定自定义域名**
   - Project Settings → Domains
   - 添加你的域名，按提示配置 DNS

### A.2 通过 Vercel CLI（进阶）

适合想从命令行直接部署的开发者。

#### 步骤

1. **安装 Vercel CLI**
   ```bash
   npm install -g vercel
   ```

2. **登录 Vercel**
   ```bash
   vercel login
   ```
   按提示选择登录方式（GitHub 推荐）。

3. **首次部署（预览环境）**
   ```bash
   cd /Users/carlshen/Documents/Project/Web_Game_01/games/2048
   vercel
   ```
   - 会询问项目配置：
     - Set up and deploy? **Y**
     - Which scope? 选择你的账号
     - Link to existing project? **N**
     - What's your project's name? `game-2048`
     - In which directory is your code located? `./`（或回车）
   - 部署完成会显示 Preview URL

4. **部署到生产环境**
   ```bash
   vercel --prod
   ```

5. **（可选）查看部署状态**
   ```bash
   vercel ls
   ```

### A.3 Vercel 部署后配置

#### 环境变量（无需）

2048 无需环境变量。最高分存 localStorage。

#### 自动部署

- 推送代码到 GitHub → Vercel 自动构建部署
- 默认 main 分支 → 生产环境
- 其他分支 → 预览环境

#### 性能监控

- Vercel Dashboard → Analytics
- 可查看访问量、构建时长、Edge 性能

## 四、方案 C：GitHub Pages 部署（零成本）

### C.1 通过 GitHub Actions 自动部署

**前置**：GitHub 账号

#### 步骤

1. **推送代码到 GitHub**（同 A.1 步骤 1）

2. **创建 GitHub Actions 工作流**

   在项目根目录创建 `.github/workflows/deploy-2048.yml`：

   ```yaml
   name: Deploy 2048 to GitHub Pages

   on:
     push:
       branches: [main]
       paths:
         - 'games/2048/**'
         - '.github/workflows/deploy-2048.yml'

   jobs:
     build-and-deploy:
       runs-on: ubuntu-latest
       defaults:
         run:
           working-directory: games/2048
       steps:
         - uses: actions/checkout@v4

         - uses: pnpm/action-setup@v4
           with:
             version: 9

         - uses: actions/setup-node@v4
           with:
             node-version: 20
             cache: 'pnpm'

         - run: pnpm install
         - run: pnpm build

         - uses: peaceiris/actions-gh-pages@v3
           with:
             github_token: ${{ secrets.GITHUB_TOKEN }}
             publish_dir: ./games/2048/dist
   ```

3. **配置 Vite base 路径**

   修改 `games/2048/vite.config.ts`：
   ```ts
   export default defineConfig({
     base: '/Web_Game_01/', // ← 改成你的仓库名
     // ...
   });
   ```

4. **启用 GitHub Pages**
   - 仓库 Settings → Pages
   - Source 选 **Deploy from a branch**
   - Branch 选 `gh-pages` / `/root`
   - 保存

5. **等待 Actions 完成**
   - 几分钟后访问：`https://<用户名>.github.io/Web_Game_01/`

## 八、常见问题

### Q1: 部署后页面空白？

A：通常是 base 路径不对。
- 检查 `vite.config.ts` 的 `base` 配置
- Vercel/Cloudflare 默认相对路径 `./` 即可
- GitHub Pages 需要改成 `/<仓库名>/`

### Q2: 静态资源 404？

A：同上，base 路径问题。
- 打开 DevTools → Network 看哪些资源 404
- 调整 `vite.config.ts` 的 `base`

### Q3: pnpm install 报 esbuild 错误？

A：pnpm 11+ 默认阻止 build script。
- 创建 `pnpm-workspace.yaml`：
  ```yaml
  onlyBuiltDependencies:
    - esbuild
  ```
- 或运行 `pnpm approve-builds` 选择 esbuild

### Q4: 第一次部署很慢？

A：Vercel/Cloudflare 首次需要安装依赖（pnpm install），约 1-2 分钟。后续会缓存，几秒即可。

### Q5: 国内访问 Vercel 慢？

A：换 Cloudflare Pages，国内访问友好。

### Q6: iOS Safari 触屏没反应？

A：检查 `index.html`：
- 包含 `<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no" />`
- 触屏事件绑在 Board DOM 节点上，绑到 window 会冲突

### Q7: 音效没声音？

A：iOS Safari 必须在用户交互后才能播放 AudioContext。
- `engine.start()` 中调 `audio.resume()` 确保是 user-gesture 后
- 首次访问需要点击屏幕或按键

### Q8: 想要自定义域名？

A：
- **Vercel**：Project Settings → Domains → 添加 + 配置 DNS
- **Cloudflare**：Pages → Custom domains → 添加
- **GitHub Pages**：Settings → Pages → Custom domain

## 九、vercel.json 详解

当前 `games/2048/vercel.json`：

```json
{
  "$schema": "https://openapi.vercel.sh/vercel.json",
  "version": 2,
  "name": "game-2048",
  "buildCommand": "npm run build",
  "outputDirectory": "dist",
  "installCommand": "npm install",
  "framework": "vite",
  "cleanUrls": true,
  "trailingSlash": false,
  "headers": [
    {
      "source": "/assets/(.*)",
      "headers": [
        { "key": "Cache-Control", "value": "public, max-age=31536000, immutable" }
      ]
    },
    {
      "source": "/(.*)\\.html",
      "headers": [
        { "key": "Cache-Control", "value": "public, max-age=0, must-revalidate" }
      ]
    },
    {
      "source": "/(.*)",
      "headers": [
        { "key": "X-Content-Type-Options", "value": "nosniff" },
        { "key": "X-Frame-Options", "value": "DENY" },
        { "key": "Referrer-Policy", "value": "strict-origin-when-cross-origin" }
      ]
    }
  ]
}
```

**关键配置**：
- `outputDirectory: "dist"`：与 Vite 默认输出对齐
- `framework: "vite"`：Vercel 自动适配
- `cleanUrls: true`：`/foo` 而不是 `/foo.html`
- `headers`：静态资源 1 年缓存，HTML 0 缓存，安全头

> **注意**：如果用 pnpm，需要把 `buildCommand` 改成 `pnpm build`，`installCommand` 改成 `pnpm install`。
